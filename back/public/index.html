Storage

>>>>IMPORTANTE: Base de datos temporaria para cargar imágenes que luego son guardadas en gallery de Professionals
    • Método: GET
    • Resultado: Trae todos los recursos disponibles en la base de datos storage.
    • Ruta: /api/v1/storage
    • Método: GET
    • Resultado: Trae un recurso de la base de datos storage que coincide con el identificador de la storage especificada en /:storageId
    • Ruta: /api/v1/storage/:storageId
    • Método: POST con headers: { 'Content-Type': 'multipart/form-data', },
    • Resultado: Crea un nuevo recurso en la base de datos storage y guarda la imagen en cloudinary
    • Ruta: /api/v1/storage
    • Body: 
              {
            "imageId": "Id que identifica la imagen con cloudinary"(String),
            "imageUrl": "Url de la imagen de cloudinary" (String),
        }
    • Método: DELETE
    • Resultado: Borra la imagen guardada en cloudinary y un recurso en la base de datos storage que coincide con el identificador de la storage especificada en /:storageI
    • Ruta: /api/v1/storage/:storageId
